## Définition

Git est un système de contrôle de version distribué créé par Linus Torvalds en 2005. Il permet de suivre les modifications apportées aux fichiers d'un projet, de collaborer à plusieurs sur un même codebase, et de revenir à des états antérieurs si nécessaire.

## Concepts fondamentaux

### <Term>Repository</Term> (dépôt)

Un <Term>repo</Term> est un espace de stockage contenant l'ensemble des fichiers d'un projet ainsi que l'historique complet de leurs modifications. Il peut être local (sur une machine) ou distant (sur un serveur).

### <Term>Commit</Term>

Un <Term>commit</Term> représente un instantané du projet à un moment donné. Chaque commit possède un identifiant unique, un message descriptif, un auteur, et une date. L'ensemble des commits forme l'historique du projet.

### <Term>Branch</Term> (branche)

Une <Term>branch</Term> est une ligne de développement indépendante. Elle permet de travailler sur une fonctionnalité sans affecter le code principal. La branche par défaut se nomme généralement "main" ou "master".

### <Term>Merge</Term> (fusion)

Le <Term>merge</Term> consiste à intégrer les modifications d'une branche dans une autre. Cette opération combine les historiques des deux branches.

### <Term>Push</Term>

Le <Term>push</Term> envoie les commits locaux vers un <Term>repo</Term> distant. Cette action synchronise le travail local avec le serveur.

### <Term>Pull</Term>

Le <Term>pull</Term> récupère les modifications du repository distant et les intègre au repository local.

## Architecture distribuée

Contrairement aux systèmes centralisés, Git est distribué. Chaque copie du repository contient l'historique complet du projet. Cette architecture permet de travailler hors ligne et offre une redondance naturelle des données.

## Plateformes Git

Git est un outil en ligne de commande (<Term>CLI</Term>). Plusieurs plateformes proposent une interface web et des fonctionnalités collaboratives :

- **<Term>GitHub</Term>** : Plateforme la plus utilisée, propriété de Microsoft
- **GitLab** : Alternative <Term>Open Source</Term> avec CI/CD intégré
- **Bitbucket** : Solution Atlassian, intégrée à Jira

## Workflow standard

1. Création d'une <Term>branch</Term> pour la nouvelle fonctionnalité
2. Développement et <Term>commit</Term>s successifs
3. <Term>Push</Term> de la branche vers le serveur distant
4. Création d'une <Term>Pull Request</Term> pour revue
5. <Term>Merge</Term> dans la branche principale après validation
6. Suppression de la branche de travail
