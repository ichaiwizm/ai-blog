Claude Code est un outil révolutionnaire qui transforme ta façon de développer. Au lieu de simplement répondre à des questions, il peut **lire ton code**, **exécuter des commandes**, et **modifier des fichiers** directement.

## Installation

L'installation dépend de ton système d'exploitation :

<PlatformContent>
<Mac>

### Installation sur macOS

La méthode recommandée est d'utiliser Homebrew :

```bash
# Installe Homebrew si nécessaire
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Installe Claude Code
brew install anthropic/tap/claude-code

# Vérifie l'installation
claude --version
```

**Alternative avec npm** :
```bash
npm install -g @anthropic-ai/claude-code
```

</Mac>
<Windows>

### Installation sur Windows

Tu peux utiliser **winget** (recommandé) ou **npm** :

```powershell
# Avec winget (Windows Package Manager)
winget install Anthropic.ClaudeCode

# Ou avec npm
npm install -g @anthropic-ai/claude-code

# Vérifie l'installation
claude --version
```

> **Note** : Si tu préfères un environnement Unix, considère d'utiliser WSL (Windows Subsystem for Linux).

</Windows>
<Linux>

### Installation sur Linux

Utilise le script d'installation officiel :

```bash
# Script d'installation
curl -fsSL https://claude.ai/install.sh | bash

# Ou avec npm
npm install -g @anthropic-ai/claude-code

# Vérifie l'installation
claude --version
```

Pour **Ubuntu/Debian**, tu peux aussi utiliser apt :
```bash
sudo apt update
sudo apt install claude-code
```

</Linux>
<WSL>

### Installation sur WSL

Dans ton terminal WSL, utilise les mêmes commandes que Linux :

```bash
# Script d'installation
curl -fsSL https://claude.ai/install.sh | bash

# Vérifie l'installation
claude --version
```

**Conseil** : Assure-toi que ton projet est dans le système de fichiers Linux (`/home/...`) et non Windows (`/mnt/c/...`) pour de meilleures performances.

</WSL>
</PlatformContent>

## Configuration initiale

Après l'installation, tu dois configurer ton API key :

```bash
# Configure ta clé API
claude config set api_key sk-ant-xxxxx

# Ou via variable d'environnement
export ANTHROPIC_API_KEY=sk-ant-xxxxx
```

## Premier projet

Navigue vers ton projet et lance Claude :

```bash
cd mon-projet
claude
```

Claude va analyser la structure de ton projet et te demander ce que tu veux faire.

### Exemples de commandes

```
> Explique-moi l'architecture de ce projet

> Ajoute une fonction de validation email dans utils.ts

> Corrige le bug dans le fichier UserAuth.tsx ligne 42

> Lance les tests et corrige les erreurs
```

## Bonnes pratiques

### 1. Donne du contexte

Plus tu donnes de contexte, meilleurs seront les résultats :

```
❌ "Corrige le bug"
✅ "Le test login.test.ts échoue avec 'TypeError: undefined is not a function'.
   Je pense que c'est lié au mock de la fonction authenticate()."
```

### 2. Vérifie les changements

Toujours vérifier ce que Claude a modifié avant de commit :

```bash
git diff
```

### 3. Utilise les commits atomiques

Laisse Claude créer des commits réguliers pour pouvoir revenir en arrière facilement.

## Conclusion

Claude Code change la donne pour les développeurs. En comprenant ton code et en pouvant agir dessus, il devient un véritable assistant de développement plutôt qu'un simple chatbot.

Commence par des tâches simples, et augmente progressivement la complexité à mesure que tu te familiarises avec l'outil.
